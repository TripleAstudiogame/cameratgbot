Документация Report Camera

Интеллектуальная система мониторинга NVR камер, парсинга Email-отчётов и Telegram-уведомлений корпоративного уровня.

Умный IMAP Парсер

Асинхронно читает почту Mail.ru, извлекает вложения (даже Apple HEIC) и выявляет типы событий.

Telegram Бот API

Рассылает фото и видео пользователям с учётом кулдаунов и прав доступа.

Безопасный Дашборд

Админ-панель на FastAPI с JWT-авторизацией, строгими заголовками CSP и HSTS.

Auto-Updater

Автоматическое обновление кода с GitHub без остановки работы и потери данных базы.

Архитектура системы

Проект построен на гибридной модели: асинхронный веб-сервер и фоновый многопоточный движок.

  • Веб-сервер: FastAPI + Uvicorn (порт 6565). Отвечает за REST API и отдачу статики.
  • Движок: engine.py. Фоновые потоки, которые каждые N секунд проверяют почту по каждой организации.
  • База данных: SQLite в режиме WAL (Write-Ahead Logging). Обеспечивает неблокирующий конкурентный доступ к данным.
  • Телеграм-интерфейс: telebot (pyTelegramBotAPI) используется в режиме polling, интегрированный в цикл проверки почты.

Типы уведомлений (Парсеры)

Система умеет распознавать три ключевых типа событий на основе темы писем NVR (Network Video Recorder):

1. Обнаружение человека

Срабатывает при темах "Person Alarm" или "Human". Формирует отчёт о проходе человека, извлекая прикреплённые снимки.

2. Обнаружение транспорта

Срабатывает при темах "Vehicle" или "License Plate". Парсер умеет находить гос.номер автомобиля в тексте письма и выводить его отдельным жирным шрифтом в Telegram.

3. Обнаружение движения

Стандартный детектор "Motion Detected". Если ИИ не определил тип объекта, фиксируется базовое движение.

Конвертация HEIC изображений

NVR устройства от Apple или определенные iOS-клиенты могут отправлять фото в формате HEIC/HEIF. Telegram не поддерживает этот формат напрямую.

Мы используем библиотеку pillow_heif совместно с Pillow. Вся конвертация происходит в оперативной памяти (io.BytesIO) "на лету" без сохранения временных файлов на диск, что сильно экономит ресурс SSD сервера.

Запуск на Windows Server 2019

Сервер работает как системная служба Windows (Service) с использованием утилиты NSSM. Это гарантирует автозапуск после перезагрузки и авторестарт в случае падения.

# 1. Открытие порта 6565 в Firewall
Запуск от Администратора: .\setup_firewall.ps1

# 2. Установка службы
Запуск от Администратора: .\install_service.bat

Автообновление (GitHub CI/CD)

Для обновления приложения вам не нужно заходить на сервер через RDP. Достаточно сделать git push origin main с вашего локального компьютера.

На сервере работает планировщик задач (Task Scheduler), который каждые 5 минут дёргает скрипт auto_updater.ps1.

Как работает автообновление:

Скрипт проверяет новые коммиты. Если они есть -> останавливает NSSM службу -> делает git reset --hard -> обновляет зависимости pip -> запускает службу. Весь даунтайм составляет менее 10 секунд.

Защита данных и Безопасность

Система спроектирована с учётом Enterprise-стандартов безопасности:

  • Маскировка данных: Токены ботов (861507***sZl4) и пароли (••••••••) никогда не отдаются на клиент в открытом виде.
  • JWT и Хеширование: Авторизация по JWT с коротким сроком жизни. Пароли в БД хешируются медленным алгоритмом PBKDF2 с уникальной солью.
  • Gitignore: Файлы данных (organizations.db, .env, bot.log) жёстко игнорируются гитом. `git reset --hard` при автообновлении никогда их не удалит.
  • Security Headers: API защищено строгими политиками: Content-Security-Policy, X-Frame-Options (DENY), HSTS. Уязвимости XSS и Clickjacking полностью исключены.
  • Rate Limiting: Форма входа защищена ограничителем 5 попыток в минуту.

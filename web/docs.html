Документация Report Camera

Полное руководство по интеллектуальному мониторингу NVR камер, автоматизированной установке и архитектуре.

Установка на Windows Server за 1 клик (1-Click Install)

Мы полностью избавились от ручной установки зависимостей (Python, Git, скрипты брандмауэра и т.д.). Теперь система устанавливает себя сама даже на абсолютно "голый" Windows Server 2019.

Всё, что вам нужно:

Просто скачайте репозиторий (или файлы проекта) на сервер и запустите файл setup_server.bat.

1

Запуск установщика

Дважды кликните по setup_server.bat. Он запросит права администратора и запустит главный скрипт развертывания.

2

Автоматическое скачивание зависимостей

Скрипт проверит наличие Python 3.11 и Git. Если их нет — он сам скачает официальные установщики из интернета и установит их в "тихом" (Silent) режиме, не требуя от вас нажимать "Далее". Также он скачает утилиту NSSM для создания системной службы.

3

Настройка окружения и Firewall

Скрипт сам откроет порт 6565 в Брандмауэре Windows, создаст виртуальное окружение venv и установит все Python-библиотеки.

4

Фоновые службы и Автообновление

Скрипт зарегистрирует приложение как системную службу ReportCamera (она будет сама стартовать при перезагрузке сервера). А также создаст невидимую задачу в Планировщике Задач, которая каждые 5 минут будет проверять GitHub на обновления.

Парсинг писем (Искусственный интеллект)

Сервер асинхронно подключается к IMAP-серверу Mail.ru и анализирует тему письма, чтобы понять, какой ИИ-триггер сработал на камере:

  • Человек: Теги Person Alarm, Human, PD. Формирует уведомление с 🧍‍♂️ иконкой.
  • Транспорт: Теги Vehicle, License Plate, Motor Vehicle. Если ИИ распознал гос.номер авто (например, [01A123BC]), парсер вырежет его из текста и выделит жирным.
  • Движение: Теги Motion, MD. Стандартный детектор движения.

Конвертация Apple HEIC "на лету"

NVR устройства, настроенные через экосистему Apple, часто присылают скриншоты в формате .heic или .heif. Telegram API не поддерживает эти форматы (он выдаст ошибку при отправке).

Решение: Модуль pillow_heif перехватывает такие файлы. Они загружаются в оперативную память (через io.BytesIO) и мгновенно конвертируются в .jpeg. Это полностью избавляет SSD диск сервера от операций записи временных файлов.

Архитектура Базы Данных (SQLite WAL)

В проекте используется встроенная БД SQLite. Чтобы предотвратить ошибку database is locked при одновременных запросах (когда фоновый движок проверяет почту, а админ в это же время редактирует настройки на сайте), мы включили режим Write-Ahead Logging (WAL).

WAL позволяет читателям (чтение БД) не блокировать писателей (запись в БД) и наоборот, обеспечивая надежную конкурентную работу.

GitHub Auto-Updater (Автообновления)

Вам больше не нужно подключаться к серверу (RDP), чтобы загрузить новые фичи.

Процесс работы:

1. Вы пишете код на своем ноутбуке и делаете git push origin main.
2. Сервер каждые 5 минут делает git fetch.
3. Если есть новый код, сервер останавливает службу, скачивает код, делает pip install и запускает службу.

Защита данных: файлы .env, organizations.db и логи защищены файлом .gitignore, поэтому автообновление их никогда не удалит.

Политики безопасности

  • Маскировка: Токены ботов (861507***sZl4) и пароли (••••••••) замаскированы в API.
  • Хеши: Пароль администратора хранится в виде PBKDF2-HMAC-SHA256 хеша (в 100 000 итераций).
  • HTTP Headers: На всех ответах сервера стоят заголовки Content-Security-Policy, X-Frame-Options: DENY, и HSTS, защищающие панель от XSS и Clickjacking.
